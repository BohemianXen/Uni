How can I implement convolution in frequency domain?

up vote 2 down vote favorite

Suppose, we have a bitmap image represented as a 2D integer array, int [,] image2D; whose FFT is Complex[,] fftImage2D;

Suppose, we have an kernel represented as a 2D integer array, int [,] kernel2D; whose FFT is Complex[,] fftKernel2D;

We know that, the convolution (in spatial domain) of image2D and kernel2D would be, 

int Rows = image2D.GetLength(0);
int Cols = image2D.GetLength(1);

for(int i=0 ; i<Rows ; i++)
{
    for(int j=0 ; j<Cols ; j++)
    {
        //sweep the kernel2D across image2D
        //...........................
    }
}

We also know that, convolution in frequency domain would be, multiplication between fftImage2D and fftKernel2D.

How can I do this multiplication?

How can I multiply two Complex [,] type 2D arrays of different dimensions? I have understood the theory. My problem is practical implementation. As I described in the question,

  1. Are DFT of the image and DFT of the kernel going to be of different sizes? I guess so. So, how can I multiply them element by element?

  2. In my code, each of the DFTs are represented by 2D Complex numbers. Should, I multiply them according to complex-number's multiplication rule? Probably yes. But, only when their dimensions are same. Right?

This (kind of) question has been possibly asked and answered a few times, but for your convenience let me put it here again:

Given a 2D input (image) sequence x[n1,n2]x[n1,n2] of length N1N1 along n1n1-axis from n1=0n1=0 to n1=N1−1n1=N1−1 and length N2N2 along n2n2-axis from n2=0n2=0 to n2=N2−1n2=N2−1, and a 2D filter's impulse response sequence h[n1,n2]h[n1,n2] of length L1L1 along n1n1-axis and length L2L2 along n2n2-axis similarly.

Then, their 2D discrete-space convolution defined with

y[n1,n2]=x[n1,n2]∗h[n1,n2]y[n1,n2]=x[n1,n2]∗h[n1,n2]
y[n1,n2]=∑∑x[m1,m2]h[n1−m1,n2−m2]y[n1,n2]=∑∑x[m1,m2]h[n1−m1,n2−m2]

has the length M1=N1+L1−1M1=N1+L1−1 along n1n1 axis starting from n1=0n1=0 and length M2=N2+L2−1M2=N2+L2−1 along n2n2 axis similary.

By rewoking the well known convolution - multiplication theorem of the DSFT (Discrete Space Fourier Transform) one can compute the result y[n1,n2]y[n1,n2] of the discrete space convolution operation in the frequency domain.

The process begins by computing the two M1×M2M1×M2 - point 2D-DFTs, X[k1,k2]X[k1,k2] and H[k1,k2]H[k1,k2] of the input sequences x[n1,n2]x[n1,n2] and h[n1,n2]h[n1,n2]. Then the resulting M1×M2M1×M2 point 2D DFTs are multiplied sample by sample to produce the the output 2D-DFT sequence Y[k1,k2]Y[k1,k2]. Finally the multiplicand Y[k1,k2]Y[k1,k2] is transformed back into space domain by an M1×M2M1×M2 point inverse DFT to produce the expected resulting sequence y[n1,n2]y[n1,n2] in a much shorter processing time when the filter's impulse response sequence length is longer than some decidable number.

Note that, the theorem which states the relation between the space and frequency domains, assumes a linear convolution between the input sequences x[n1,n2]x[n1,n2] and h[n1,n2]h[n1,n2]. hHowever its practical application is based on the DFT sequences and muliplication between two DFT sequences equals circular convolution of them in their in space domains. Therefore to equate the linear convolution assumed by the theory to the circular convolution provided by the practical DFT, the lengths of the intermediate DFT sequences should all be selected properly to avoid any time-aliasing during the inverse DFT stage. The provided M1M1 and M2M2 are the minimum values for the exact computation of y[n1,n2]y[n1,n2].

Implementing the above in matlab/octave can be done with the following line:

Iy = real( ifft2( fft2(Ix, M2,M1).*fft2(h,M2,M1) , M2, M1) );

It's important to recognize the fact that eventhough the theory above uses the sequence notation f[n1,n2]f[n1,n2] to describe the input and output signals and operations performed on them, MATLAB uses matrices to represent signals and perform operation on them. The default mapping between the elements of the matrix Ix(i,j)Ix(i,j) and the samples of the sequence x[n1,n2]x[n1,n2] is given by the following index mapping:

Ix(i,j)=x[n1,n2]|n1=j−1 , n2=i−1Ix(i,j)=x[n1,n2]|n1=j−1 , n2=i−1

The mapping is such that the origin sample x[0,0]x[0,0] of the sequence is mapped to the first element Ix(1,1)Ix(1,1) of the matrix Ix(i,j)Ix(i,j). This is basicaly a flipped-down reordering of the sequence samples x[n1,n2]x[n1,n2].

Based on yor intention of the processing, you can retain the whole of output sequence or select a portion of it such as the center for typical image filtering applications.

IDFT the smaller or both of the DFTs if needed. Zero pad one or both of the kernel and image to make them the same dimension and size. Re-DFT as needed, and now you can complex multiply the 2 DFT arrays element-by-element because they will now be the same size.
